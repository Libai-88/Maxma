# MaxmaHere 当前架构

> 本文档是项目现行架构的唯一总览。内容以当前代码、构建脚本和测试为准，不记录迁移过程，也不保留已删除实现的操作说明。
>
> 更新时间：2026-07-23

## 1. 项目定位

MaxmaHere 是一个本地优先的 AI Agent 桌面客户端。Maxma 负责桌面体验、用户配置、会话管理、前后端通信和安全边界；oh-my-pi 负责 Agent 推理循环、模型调用和工具执行。

当前 Agent 执行路径只有一条：

```text
Vue / Tauri WebView
        |
        | HTTP + WebSocket
        v
FastAPI 后端
        |
        | JSON-RPC 2.0 over stdio
        v
Bun sidecar
        |
        | createAgentSession
        v
oh-my-pi Agent
        |
        +-- LLM Provider
        +-- 内置工具
        +-- Maxma TypeScript 工具
        +-- MCP 服务器
```

不要在 Python 后端重新实现 Agent 循环，也不要根据旧文档寻找已删除的图、节点、检查点或 Python Tool 基类。

## 2. 四层职责

### 2.1 Vue 前端：`web/`

入口：`web/src/main.ts`、`web/src/App.vue`

职责：

- 页面、路由、主题和设计系统
- Pinia 状态管理
- REST API 调用和 Token 获取
- 每个会话独立的 WebSocket 连接
- token、工具、审批、计划和错误事件的渲染
- 浏览器开发模式与 Tauri 桌面模式的 URL/端口适配

关键位置：

- `web/src/router/index.ts`：页面路由
- `web/src/api/index.ts`：REST 客户端
- `web/src/composables/useChat.ts`：聊天 WebSocket 生命周期
- `web/src/stores/session.ts`：会话列表和当前会话
- `web/src/stores/chat.ts`：消息、流式状态和上下文用量
- `web/src/components/tools/registry.ts`：工具结果组件路由

### 2.2 Python 后端：`api/`

入口：`main.py` -> `api/server.py:create_app()`

职责：

- FastAPI 应用生命周期和路由注册
- HTTP Token 鉴权、WebSocket 鉴权、CORS 和限流
- Maxma session 生命周期和 REST CRUD
- 前端 WebSocket 与 sidecar JSON-RPC 之间的事件桥接
- Provider、MCP、Skill、Macro、人设和运行时配置持久化
- 路径白名单、MaxmaBlocker、凭据加密等 Maxma 安全策略
- 生产模式下静态文件服务

Agent 对话的核心后端入口是 `api/routes/chat.py`。它只负责准备模型配置、建立 sidecar session、转发事件和保存会话数据。

### 2.3 Bun sidecar：`bun-sidecar/`

入口：`bun-sidecar/src/session-bridge.ts`

职责：

- 启动并维护 oh-my-pi `AgentSession`
- 解析 provider/model 配置
- 注册 Maxma TypeScript 工具
- 加载并连接 MCP 服务器
- 执行 prompt、cancel、undo、compact 和消息读取
- 将 oh-my-pi 事件映射为 Maxma 事件
- 在 ask/read_only 权限下等待前端审批

Python 通过 `api/pi_bridge/rpc_client.py` 发送 JSON-RPC 请求，sidecar 通过 stdout 发送响应和 `event` 通知。

当前主要 RPC 方法：

```text
create_session
prompt
cancel
destroy_session
undo
compact
get_messages
get_health
user_response
reload_mcp
```

`reload_mcp` 当前返回“需要重建 session”，修改 MCP 配置后不要假设现有 Agent session 已经刷新。

### 2.4 Tauri 桌面壳：`desktop/src-tauri/`

入口：`desktop/src-tauri/src/main.rs`

职责：

- 启动 PyInstaller 后端 sidecar
- 选择可用本地端口并通过 Tauri command 提供给前端
- 等待 `/api/health` 就绪
- 注入资源目录、父进程 PID 和运行时环境变量
- 使用 Windows Job Object 清理后代进程
- 处理 sidecar 崩溃重启、WebView2 检查、全局快捷键和原生文件对话框

桌面前端必须使用 `tauriFetch()` 访问本地 HTTP API。不要在 Tauri 页面中直接用原生 `fetch()` 访问 `http://127.0.0.1`。

## 3. 一次对话的生命周期

1. 前端通过 `ensurePortLoaded()` 获取 Tauri 选择的 API 端口。
2. 前端通过 `/api/auth/token` 获取运行时 Token。
3. 前端连接 `/ws/chat/{session_id}`，Token 放入 WebSocket subprotocol。
4. FastAPI 校验 Token，获得 `SessionState`，并启动或复用 `SidecarManager`。
5. `SessionMap` 查询 Maxma session 到 sidecar session 的映射。
6. 映射不存在或已失效时，后端调用 `create_session`，传入 provider、model、system prompt、项目根目录和权限模式。
7. 用户消息通过 `prompt` RPC 进入 oh-my-pi。
8. sidecar 把 token、thinking、tool、ask_user、plan、answer、done 和 error 事件发送给 Python。
9. Python 将事件转发到当前 WebSocket；前端更新对应 Pinia channel。
10. 回合完成后，Python 更新消息计数、SessionMap 摘要和固定会话数据；前端保存可恢复的 UI turn 缓存。

取消和断线必须经过 sidecar `cancel` RPC。不要只取消 Python asyncio task，否则 Agent 可能继续运行并消耗模型配额。

## 4. 状态与持久化

### 4.1 会话

- 内存会话：`api/session_manager.py`
- Maxma 与 sidecar 映射：`api/pi_bridge/session_adapter.py`
- 映射数据库：开发模式下为用户目录 `.maxma/session_map.db`，冻结模式迁移到应用数据目录
- sidecar 消息：由 oh-my-pi session 管理
- 前端 UI turn：`localStorage`，键名为 `maxma_turns_<session_id>`
- 固定会话：`api/const_session_store.py` 和 `api/data/const-sessions/`

SessionMap 是映射和最近对话摘要缓存，不是 Agent 的完整消息事实源。需要完整消息时通过 sidecar `get_messages` 获取。

### 4.2 用户数据路径

路径解析集中在 `app_paths.py`：

| 场景 | 资源 | 可写数据 |
| --- | --- | --- |
| 开发模式 | 项目根目录 | 项目根目录 |
| PyInstaller/Tauri | bundle/resource 目录 | `%APPDATA%\\MaxmaHere` |

常用可写数据包括：

- `api/data/providers.yaml`
- `api/data/mcp_servers.yaml`
- `api/data/auth_token.yaml`
- `api/data/maxma.db`
- `api/data/path_whitelist.yaml`
- `api/data/maxma_blocker.yaml`
- `config/personas/`
- `anthropic_skills/`
- `macros/`
- `uploads/`
- `logs/`
- `vector_db/`

不要把用户运行时数据递归打进 PyInstaller bundle。构建 spec 只应包含内置模板和运行必需资源。

## 5. Provider、工具、Skill 和 MCP

### Provider

Provider 配置由 `api/routes/providers.py` 管理，存储在 `api/data/providers.yaml`。API Key 使用凭据信封加密，前端只拿到脱敏结果。Provider 的 `base_url` 是用户输入的网络目标，保存和请求前都必须经过 URL 校验。

### 工具

内置 Agent 工具由 oh-my-pi 提供。Maxma 工具在 `bun-sidecar/src/tools/` 定义并由 `registerCustomTools()` 注册，当前包括天气、节假日、塔罗牌、Todoist 以及若干配置管理工具。新增工具时需要同时考虑：

- TypeScript `ToolDefinition` 和 zod 参数
- 工具执行超时和错误返回
- `api/routes/chat.py` 的事件转发
- 前端 `web/src/components/tools/` 的展示组件
- 权限、路径和凭据边界

### Skill 和 Macro

- 内置 Skill：`anthropic_skills/`
- 用户 Skill：由后端写入应用数据目录
- Macro：`macros/` 和对应配置路由

Skill 是 Agent 可加载的行为扩展，不是 Python 路由，也不是 Tauri 插件。

### MCP

MCP 配置由 `api/routes/mcp.py` 管理，sidecar 在创建 Agent session 时读取 `api/data/mcp_servers.yaml` 并连接支持的 transport。stdio 命令必须通过 Maxma 白名单校验。修改 MCP 配置后，新建 session 才能保证使用最新配置。

## 6. 安全边界

安全边界以 [security-contract.md](security-contract.md)、[dev_docs/permission-modes.md](../dev_docs/permission-modes.md)、[dev_docs/path-whitelist.md](../dev_docs/path-whitelist.md) 和代码为准。

必须保持的原则：

- 空路径白名单默认拒绝访问
- 路径解析失败、符号链接越界和 blocker 检查异常都 fail-closed
- Token 不写入前端构建产物
- API Key 不写入日志、事件或前端响应
- Tauri 本地请求必须经过 `tauriFetch()`
- MCP stdio 命令只能执行白名单内容
- Provider URL 必须拒绝危险协议和云元数据地址
- Python/Bun 子进程必须继承受限的环境变量和本地代理绕过配置

## 7. 开发、测试与构建

### 开发

```text
后端：python main.py 或 start_dev.py
前端：cd web && npm run dev
桌面：build\\run-desktop-dev.bat
```

### 测试

```text
Python：pytest -q
前端：cd web && npx vitest run
Sidecar：cd bun-sidecar && bun test
```

### 构建

```text
服务端：build\\build-server.bat
桌面安装包：build\\build-desktop.bat
构建契约：powershell -NoProfile -ExecutionPolicy Bypass -File build\\test-build-contract.ps1
```

服务端构建顺序是前端构建、Bun 准备与依赖安装、PyInstaller 打包、后端 smoke test、部署到 Tauri sidecar 目录。不要用开发目录中的用户配置替代构建资源。

# 翻译工作流

把一种语言的文本翻译成另一种语言，**重点是"信达雅"而不是逐字对应**。

## 适用场景

- 用户给一段文字 / 文档，要求翻译
- 用户要把技术文档译成中文（或英文）
- 用户要本地化软件界面 / README

## 工作流程

### 1. 确认翻译要求

用 `ask_user_qa` 一次问清：

1. **源语言 → 目标语言**：从什么语言译到什么语言？
2. **文本类型**：技术文档 / 文学 / 商务 / 日常对话 / UI 文案？
3. **风格**：正式 / 半正式 / 口语化？
4. **术语处理**：专业术语保留原文（带括号注释）/ 完全本地化 / 行业惯用译法？
5. **格式保留**：是否需要保留 Markdown 格式 / 代码块 / 链接？

### 2. 读取原文

- 本地文件：用 `file_read` 读取
- 知识库：用 `kb_search` 检索
- 用户口述：直接在对话里

### 3. 翻译策略

按文本类型选择策略：

| 文本类型 | 策略 |
|---------|------|
| 技术文档 | 术语优先用行业惯用译法（如 "token" 不译为"代币"而保留）。代码、命令、路径、配置项**不翻译**。 |
| 文学 | 注重语言美感，可以重组句子结构，保留原文意境。 |
| 商务 | 正式、简洁、避免歧义。数字 / 日期 / 货币按目标语言习惯格式化。 |
| UI 文案 | 简短（受界面空间限制），动词前置，避免被动语态。 |
| 日常对话 | 口语化，可以用俚语 / 网络用语（如果原文也用了）。 |

### 4. 执行翻译

**翻译原则**：

1. **理解再重写**：先完整理解原文意思，再用目标语言"重新写一遍"，不是逐字替换。
2. **保留结构**：原文的段落、列表、标题层级、代码块、链接都要保留。
3. **术语一致**：同一术语全文统一译法，不要一会 "服务器" 一会 "服务端"。
4. **数字格式**：
   - 中文：1,234.56 → 1,234.56（保留西式）
   - 中文大写：1,234 → 一千二百三十四（仅法律 / 财务文件）
   - 日期：Jul 4, 2026 → 2026年7月4日
5. **文化适配**：
   - 比喻：英文 "raining cats and dogs" → 中文 "倾盆大雨"（不是"下猫下狗"）
   - 计量单位：mile → 公里（标注换算）
   - 人名 / 地名：保留原文 + 首次出现时附中文译名
6. **不要过度翻译**：
   - 代码标识符（变量名、函数名）不译
   - 文件路径不译
   - 命令行不译
   - URL 不译
   - 品牌名 / 产品名通常不译（如 "GitHub" 不译为"吉特哈布"）

### 5. 自检

翻译完通读一遍，检查：

- [ ] 有没有漏译段落？
- [ ] 术语是否全文一致？
- [ ] 数字 / 日期 / 单位是否按目标语言习惯格式化？
- [ ] 代码块 / 命令 / 路径是否保留原样？
- [ ] 语气是否与原文一致？
- [ ] 有没有"翻译腔"（生硬的逐字对应）？

### 6. 交付

- 短文本：直接在对话里给出译文
- 长文档：用 `file_write` 保存到文件（默认 `{原文件名}_zh.md`）
- 提供"译者注"解释关键翻译决策（如术语选择、文化适配）

## 常见翻译陷阱

| 陷阱 | 错误 | 正确 |
|------|------|------|
| 直译成语 | "It's a piece of cake" → "这是一块蛋糕" | → "小菜一碟" |
| 术语错译 | "Token (NLP)" → "代币" | → "词元" 或保留 "token" |
| 被动语态滥用 | "The function is called by..." → "函数被...调用" | → "...调用该函数" |
| 长定语从句 | "The system that was developed by X in 2020..." → 直接照搬结构 | → 拆成两句 |
| 代词指代不清 | "它" "它" "它" | → 明确指代对象 |
| 否定前移 | "I don't think he is right" → "我不认为他是对的" | → "我认为他不对" |

## 双语对照（可选）

如果用户要学习用途，可以提供双语对照：

```markdown
| 原文 | 译文 |
|------|------|
| The quick brown fox jumps over the lazy dog. | 敏捷的棕色狐狸跳过懒狗。 |
```

## 注意事项

- **不要用机器翻译腔**：翻译出来要像目标语言母语者写的，不是"中文皮英文骨"。
- **保留代码原样**：代码块、命令行、文件路径、配置项一律不译。
- **术语要查证**：不确定的专业术语，用 `tavily_search` 查行业惯用译法。
- **数字不要四舍五入**：3.14159 不要译成 3.14，除非用户要求。
- **保密**：翻译的合同 / 内部文档内容不要回显到对话里。
- **不要润色过度**：翻译是"忠实传达"，不是"改写"。原文啰嗦，译文也啰嗦（除非用户要求精简）。

## 推荐工具组合

| 阶段 | 主用工具 |
|------|---------|
| 读原文 | `file_read` / `kb_search` |
| 澄清要求 | `ask_user_qa` |
| 术语查证 | `tavily_search` |
| 写译文 | `file_write` / `file_edit` |
| 入库 | `kb_add_document`（可选） |

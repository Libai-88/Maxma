# 学习笔记整理工作流

把原始学习材料（文章、视频字幕、书本章节、课程内容）转化为结构清晰、可检索、可复习的笔记。

## 适用场景

- 用户读完一篇文章 / 看完一个视频，想做笔记
- 用户在学习一门课程，想系统整理每节课的要点
- 用户想把零散的学习记录归并成一份系统笔记

## 工作流程

### 1. 确定材料来源

材料来源可能是：

- **本地文件**：用 `file_read` 读取（`.md` / `.txt` / `.pdf`）
- **知识库已有**：用 `kb_search` 检索
- **网页 URL**：用 `tavily_extract` 抓取正文
- **用户口述**：用户直接在对话里贴内容

用 `ask_user_qa` 确认：
1. 材料来源（上述哪种）
2. 学习目的（考试 / 工作应用 / 个人兴趣 / 教别人）
3. 笔记保存位置（默认 `D:\Users\<user>\Documents\notes\<topic>.md`）

### 2. 阅读与提炼

通读材料后，提炼以下结构：

```markdown
# {主题}

## 元信息
- 来源：[作者/出处](url)
- 学习日期：YYYY-MM-DD
- 学习目的：……

## 核心要点（3–5 条）
1. ……
2. ……

## 详细笔记
### 概念 A
- 定义：……
- 例子：……
- 易混淆：……vs……

### 概念 B
……

## 疑问 / 待深入
- ……

## 与已有知识的关联
- 关联到 [另一篇笔记]：……

## 应用场景
- ……
```

### 3. 关联已有知识

- 用 `kb_search` 搜相关主题，看知识库里是否已有相关笔记
- 用 `search_memories` 搜长期记忆里的相关内容
- 在"与已有知识的关联"章节明确引用：`关联到 [已有笔记标题]`

> 这一步是学习笔记的关键——**孤立的知识容易忘，关联的知识记得牢**。

### 4. 生成可复习的卡片（可选）

如果用户学习目的是"考试"或"长期记忆"，额外生成 Anki 风格的问答卡片：

```markdown
## 复习卡片
Q: 什么是 X？
A: ……

Q: X 和 Y 的区别？
A: ……
```

### 5. 保存与入库

1. 用 `file_write` 保存 Markdown 笔记到用户指定位置
2. 询问用户是否同步入库（`kb_add_document`），section 填 `学习笔记`
3. 入库后立即 `kb_search` 自检，确认能搜到
4. 关键概念用 `save_memory` 存为长期记忆（如"用户正在学习 X，掌握程度 Y"）

## 不同材料的处理策略

| 材料类型 | 处理要点 |
|---------|---------|
| 学术论文 | 重点：研究问题 / 方法 / 结论 / 局限。参考 `paper-reproduction` skill |
| 技术文章 | 重点：解决什么问题 / 怎么做 / 代码示例。保留可运行代码片段 |
| 视频/课程 | 重点：分段标题 + 每段要点。时间戳标注 |
| 书籍章节 | 重点：章节逻辑脉络 + 关键概念。画出概念关系 |
| 官方文档 | 重点：API 签名 / 配置项 / 常见错误。保留官方示例链接 |

## 注意事项

- **不要只是摘要**：摘要 ≠ 笔记。笔记要有自己的理解、关联、疑问。
- **用自己的话复述**：不要复制原文，用自己的话重述才说明理解了。
- **保留出处**：每条要点标注来源（页码 / 时间戳 / 段落），方便回查。
- **诚实标注疑问**：没看懂的地方明确标"待深入"，不要装懂。
- **不要过长**：单篇笔记控制在 500–800 字。太长说明没提炼，考虑拆分。
- **代码示例保留**：技术学习材料里的代码片段必须原样保留，不要"复述"代码。

## 推荐工具组合

| 阶段 | 主用工具 |
|------|---------|
| 读材料 | `file_read` / `kb_search` / `tavily_extract` |
| 澄清目的 | `ask_user_qa` |
| 关联已有知识 | `kb_search` / `search_memories` |
| 写笔记 | `file_write` |
| 入库 | `kb_add_document` |
| 存记忆 | `save_memory` |
